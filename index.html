📖 マニュアル
1 Metaアカウントが変わった場合
1
新しいアクセストークンを発行する
Meta Business Suite → 設定 → システムユーザー → 「トークンを生成」をクリック
2
連携設定を更新する
このツール → 連携設定 → Meta → トークンとアカウントIDを新しいものに書き換え → 保存
3
GASコードを再生成する
連携設定 → GASコード生成 → コードをコピー → GASに貼り直す
4
動作確認
GASを手動実行してSlackにデータが届くか確認する
注意： Metaのアクセストークンは約60日で期限切れになります。期限切れの場合は同じ手順でトークンを再発行してください。
2 Googleアカウント（GA4 / GTM）が変わった場合
1
新しいGA4プロパティIDを確認する
GA4管理画面 → プロパティ設定 → プロパティID（数字9桁）
2
サービスアカウントを新プロパティに追加する
GA4管理画面 → プロパティ → アクセス管理 → サービスアカウントのメールアドレスを追加（閲覧者権限）
3
連携設定のプロパティIDを更新する
このツール → 連携設定 → GA4/GTM → 新しいプロパティIDを入力 → 保存
4
GTMタグを確認する
GTMコンテナID（GTM-XXXXXXX）が変わった場合はSquadBeyondのJavaScript HeadのGTMコードも更新する
サービスアカウントメール： id-gtm-ga4-operator@gtm-kaya-setup.iam.gserviceaccount.com
このメールをGA4の新プロパティに追加することでAPIアクセスが可能になります。
3 LINE広告など他媒体を追加する場合
1
LINE広告のAPIトークンを取得する
LINE Ads Manager → アカウント設定 → API連携 → チャネルアクセストークンを発行
2
連携設定に追加する
このツール → 連携設定 → LINE広告 → トークンとアカウントIDを入力 → 保存
3
GASコードを再生成する
連携設定 → GASコード生成 → LINE項目が自動で追加されたコードが生成される
4
スプレッドシートにシートを追加する
GASを実行すると「LINE広告」シートが自動作成され、Meta・GA4と並んでデータが入る
4 キャンペーン別・記事別の計測設定（GTM + GA4）
全体の流れ： 広告URL（UTM付き）→ GTMが取得 → GA4イベントに付与 → スプレッドシートに集計
STEP 1｜UTMパラメーターを設計する
1
命名ルールを統一する
utm_source：媒体名（meta / line / google）
utm_campaign：キャンペーン名（例：kaya_spring_2026-04）
utm_content：記事パターン（例：article-a / article-b）
2
連携設定 → UTM生成ツールでURLを作成
記事URL・媒体・キャンペーン名・記事パターンを入力してURLを生成。そのURLを広告の遷移先に設定する。
STEP 2｜GTMでGA4タグを設定する
1
GTMで変数を作成する
GTM管理画面 → 変数 → 新規 → URLパラメーター変数を4つ作成
変数名：UTM Source / UTM Campaign / UTM Content / Page Path
2
GA4イベントタグを作成する
GTM → タグ → 新規 → Googleアナリティクス：GA4イベント
イベント名：lp_view
パラメーター追加： campaign = {{UTM Campaign}} / article = {{UTM Content}} / source = {{UTM Source}} / page = {{Page Path}}
3
トリガーを設定してパブリッシュ
トリガー：ページビュー（全ページ）
GTM右上「公開」→ バージョン名を入力して保存
STEP 3｜GA4でカスタムディメンションを登録する
1
GA4管理画面を開く
GA4 → 管理（左下の歯車）→ プロパティ → カスタム定義 → カスタムディメンション → 「作成」
2
3つのディメンションを登録する
① ディメンション名：campaign / スコープ：イベント / イベントパラメーター：campaign
② ディメンション名：article / スコープ：イベント / イベントパラメーター：article
③ ディメンション名：source / スコープ：イベント / イベントパラメーター：source
3
24〜48時間待つ
登録後すぐには反映されません。翌日以降にGA4 → 探索レポートで確認できます。
STEP 4｜スプレッドシートで確認する
1
GASコードを生成して貼り付ける
連携設定 → GASコード生成 → コピー → script.google.com に貼り付け → 保存
2
スプレッドシートの構成
自動で以下のシートが作られます：
「日次サマリー」：キャンペーン × 記事 × 日付 × PV / CV / CVR（最新が上）
「キャンペーン別」：キャンペーンごとの累計
「記事比較」：A記事 vs B記事 vs C記事の横並び比較
注意： GTMのタグ設定後、広告URLにUTMパラメーターがついていないとGA4にデータが入りません。必ず連携設定のUTM生成ツールでURLを作ってから広告に設定してください。
5 Atomoi Popupの運用フロー
1
プロジェクトを作成する
Popup Generator → プロジェクト名を入力（例：カヤ_LP_2026-04）
2
画像・CTA・表示設定をする
ポップアップ画像URLまたはファイルをアップロード → クリック先URLを入力 → 表示回数を設定
3
コードを生成して貼り付ける
「コードを生成する」→ コピー → SquadBeyondのJavaScript Body末尾に貼り付け → 保存
4
GA4でポップアップの効果を計測する
GA4 → イベント → atm_shown（表示数）・atm_click（CTA）をプロジェクト名でフィルタ
